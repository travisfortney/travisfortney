# Travis Fortney

I work in support operations and like building small tools that make repetitive work easier to see, check, or finish.

Most of my public projects are intentionally small. I use them to practice clean workflows: clear READMEs, simple tests, useful commit history, and enough polish that the repo explains itself without a tour guide.

## Current Little Projects

- [support-health-formula](https://github.com/travisfortney/support-health-formula) - a synthetic support ops scoring model with sample data and tests
- [done-ding](https://github.com/travisfortney/done-ding) - a tiny macOS chime for long-running terminal tasks
- [larry-david-desktop-pet](https://github.com/travisfortney/larry-david-desktop-pet) - a small desktop pet package based on my cat

## What I Like Working On

- support workflow tools
- practical automation
- small macOS and terminal helpers
- readable docs and boring checks that catch obvious mistakes

I am not trying to make everything a platform. I mostly like tools that do one job, stay understandable, and save a little future annoyance.
